## Documentation Structure
The documentation is organized to reflect a typical developer journey:

1. Start with the Quick Start Guide  
2. Learn authentication and basic requests  
3. Explore API behavior through tutorials and examples  
4. Reference detailed endpoint documentation  
5. Handle responses and errors safely  
6. Prepare for real-world usage and troubleshooting
